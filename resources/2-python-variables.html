Object Definition

All data in a Python program is represented by objects or by relations between objects.

Every object has three fundamental properties:

Identity

A unique ID that never changes (memory address)

id(3)
# 4358499512

Type

Determines valid operations, never changes

type(3)
#

Value

The actual data - may be mutable or immutable

x = 3
# value is 3

Value Mutability

An object's value can be:

Immutable

Value cannot change after creation

int
float
string
tuple
x = "hello"
y = x.upper()
# x is still "hello"
# y is new object "HELLO"

Mutable

Value can be changed in place

list
dict
set
x = [1, 2, 3]
x.pop()
# x changed to [1, 2]
# Same object, new value!
⚠️ Important: Identity and type never change, but value can change if the object is mutable!

Identity Operations

# The id() function returns an object's identity
id(3)          # returns 4358499512
id("hello")    # returns 4340502768

# The 'is' operator compares identity
id(3) is not id("hello")    # True
id(3) is not id(3)          # True (each call creates new id)
id("hello") is not id("hello")  # True
Use is and is not to check if two variables refer to the same object (same identity)

Type

An object's type determines the operations you can perform on it.

type(3)              # 
type(3.5)            # 
type("hello")         # 
type([1, 3, 5])       # 
type(True)           #
✓ Valid operations for string: 
len("hello")  # 5
"hello".upper()  # "HELLO"
✗ Invalid for int: 
len(3)      # Error!
3.upper()    # Error!

Variable

A Python variable is a symbolic name that is a reference to an object.
x
→
42

Variable x is a label that points to object 42

Technically

• A variable is a name of a memory address

• The memory content is the address of an object

Conceptually

• A variable is a named reference (label) to an object

• Variables can refer to different objects over time

A Variable Is NOT

⚠️ A variable is NOT a container/box/location!

In Python, a variable is a symbolic name/label that you bind to objects.

❌ Wrong Mental Model

"A variable is a box that contains a value"

This is true for languages like C/C++, but NOT Python!

✓ Correct Mental Model

"A variable is a label attached to an object"

Multiple labels can point to the same object!

Assignment Statement

Create a variable using: name = something

PI = 3.1415926
cat_names = ["lemon", "kiwi", "orange"]

number_input = input("Please input a number: ")
number = int(number_input)

# Assign one value to multiple variables
x = y = z = 0
⚠️ Rules:
  • LHS (Left Hand Side) must be a variable name
  • RHS (Right Hand Side) can be any object or operation
  • Must create a variable before using it (except on LHS)

Why Variable?

Named references are essential for:

1. Meaningful Names

# Bad
x = 10 ** 2 * 3.14

# Good
PI = 3.14
diameter = 10
area = diameter**2 * PI

2. Reuse in Multiple Places

names = ["Alice", "Bob"]

names.append("Carol")
print(names)
for name in names:
    print(name)

3. Runtime Data

# User input at runtime
num = input("Number: ")

# Function results
result = calculate(num)

Python Identifier

Rules for valid variable names:

✓ Valid Identifiers

score = 90
first_name = "Alice"
_private = 42

✗ Invalid Identifiers

Best Practices:
  • Use snake_case for variables and functions
  • Use UPPERCASE for constants
  • Choose meaningful names in context

Bind and Alias

When RHS is a literal:

numbers1
→
[1, 3, 5]

Variable is bound to a new object

When RHS is a variable:

numbers1
→
[1, 3, 5]
←
numbers2

Creates an alias - both refer to same object!

numbers1 = [1, 3, 5]  # binds to a list object
numbers2 = numbers1    # creates an alias
name = "Hello"          # binds to a string object

More Alias Examples

x = 3
y = 5
print(x is y)      # False - different objects
print(x is not y)  # True

z = x
print(x is z)      # True - they are aliases of 3!

⚠️ Danger with Mutable Data!

scores = [90, 67, 82]
scores2 = scores      # Creates an alias!

scores2[0] = 77

print(scores2)      # [77, 67, 82]
print(scores)       # [77, 67, 82] - Same object!

None

Python has a special type NoneType with a single value: None

It means "no value" or "missing value"

x = 3
y = None

x is None         # False
x is not None     # True
y is None         # True
y is not None     # False

type(y)             #
✓ Best Practice: Always use is or is not to check for None, not ==

Tuple

Tuple: An immutable sequence of values (like an immutable list)

# Values can be different types
my_tuple = 1, 3, "Hello"

# Single value - trailing comma is mandatory!
single_tuple = 7,

# Empty tuple
empty_tuple = ()

# Nested tuples
nested_tuple = "look", my_tuple, (3.5, 4.2)

# Access with index (0-based)
print(my_tuple[2])      # "Hello"
print(nested_tuple[1])  # (1, 3, 'Hello')

Tuple Operations

Tuples are immutable - only read-only operations allowed

my_tuple = 1, 3, "Hello"
single_tuple = 7,

# Length
print(len(my_tuple))   # 3

# Concatenation creates new tuple
combined = my_tuple + single_tuple
print(combined)        # (1, 3, 'Hello', 7)

# Unpacking - create multiple variables
number1, number2, message = my_tuple
print(number1, message)  # 1 Hello

# Use _ to ignore elements
_, _, message = my_tuple
print(message)            # Hello

Mutability

Immutable Types

Cannot be changed after creation

int
float
string
tuple
name = 'Alice'
upper = name.upper()

# name is still 'Alice'
# upper is new: 'ALICE'

Mutable Types

Can be changed in place

list
dict
set
names = ['Alice']
names.append('Bob')

# names changed!
# ['Alice', 'Bob']
⚠️ Critical: Knowing mutability is essential to avoid hard-to-find bugs!

Copy Data

Sharing mutable data can be dangerous - make a copy to be safe! 
list1 = [1, 2, 3]
list2 = list1           # Alias - same object!
list3 = list1.copy()    # Copy - new object!

list1[2] = 'three'

print(list1)  # [1, 2, 'three']
print(list2)  # [1, 2, 'three'] - Changed!
print(list3)  # [1, 2, 3] - Safe!

list1 and list2 share the same object, list3 is independent

Shallow vs Deep Copy

⚠️ Shallow Copy Problem

list1 = [1, 2, [3, 4]]
list3 = list1.copy()     # Shallow copy

list1[2].append(5)

print(list1)  # [1, 2, [3, 4, 5]]
print(list3)  # [1, 2, [3, 4, 5]] - Nested list shared!

✓ Deep Copy Solution

import copy

list1 = [1, 2, [3, 4]]
list3 = copy.deepcopy(list1)  # Deep copy!

list1[2].append(5)

print(list1)  # [1, 2, [3, 4, 5]]
print(list3)  # [1, 2, [3, 4]] - Safe!

Mutable Data as Function Argument

⚠️ The Problem

def test(numbers):
    numbers[2] = 'three'  # Modifies input!
    print(numbers)

list1 = [1, 2, 3]
test(list1)
print(list1)  # [1, 2, 'three'] - Changed outside function!

Changing input data makes code hard to understand and can introduce bugs!

✓ The Fix

import copy

def test(numbers):
    copied = copy.deepcopy(numbers)  # Work on copy!
    copied[2] = 'three'
    print(copied)

list1 = [1, 2, 3]
test(list1)
print(list1)  # [1, 2, 3] - Safe!

Summary

Key Concepts to Master:

About Objects

  • Type and identity are fixed once created
  • Type determines operations and mutability
  • Value can change if object is mutable

About Variables

  • Variables are references/labels to objects
  • Can refer to different objects over time
  • Assignment creates bindings or aliases
⚠️ Critical Warnings:
  • Be careful with mutable data - aliases can cause unexpected changes
  • Use copy.deepcopy() for nested mutable data
  • Avoid modifying function arguments in place
  • Use is/is not for identity, ==/!= for value comparison
