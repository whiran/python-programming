üéØ Match Statement (Python 3.10+)

Pattern Matching for Cleaner Code

üÜï New Feature: The match statement provides a cleaner way to handle multiple specific values, similar to switch/case in other languages.

# HTTP status code handler
def http_error(status):
    match status:
        case 200:
            return "OK ‚úÖ"
        case 400:
            return "Bad request ‚ö†Ô∏è"
        case 404:
            return "Not found üîç"
        case 418:
            return "I'm a teapot ‚òï"
        case _:  # wildcard (default)
            return "Something's wrong ü§î"

print(http_error(404))  # Output: Not found üîç

üí° The underscore (_): Acts as a wildcard that matches everything. It's like the "else" clause.

üö¶ Sentinel Loop Pattern

Using a Special Value to Stop

üéØ Real-World Analogy: Like a buffet line - you keep taking food until you see the "end of buffet" sign (the sentinel value).

# Calculate total of user inputs
# User enters -1 to stop
total = 0
number = int(input('Enter a positive number (-1 to stop): '))

while number != -1:  # -1 is the sentinel value
    total += number
    number = int(input('Enter a positive number (-1 to stop): '))

print(f'The total is {total}')

üí° Sentinel Value: A special value that signals the end of input. Common choices: -1, 0, empty string, "quit"

ü™Ü Nested Loops

üéØ Real-World Analogy: Like organizing a deck of cards - outer loop for suits (‚ô†Ô∏è‚ô•Ô∏è‚ô¶Ô∏è‚ô£Ô∏è), inner loop for ranks (A, 2, 3...K).

‚Ü©Ô∏è Return Values

Getting Results from Functions

# Function that returns a value
def add_numbers(a, b):
    """Add two numbers and return the result"""
    result = a + b
    return result

# Use the returned value
sum_value = add_numbers(5, 3)
print(f'The sum is: {sum_value}')  # Output: The sum is: 8
# Return multiple values (as a tuple)
def calculate(a, b):
    """Return sum, difference, and product"""
    return a + b, a - b, a * b

# Unpack the returned values
sum_val, diff_val, prod_val = calculate(10, 3)
print(f'Sum: {sum_val}, Diff: {diff_val}, Product: {prod_val}')
# Output: Sum: 13, Diff: 7, Product: 30

üí° Important: Functions without a return statement automatically return None

üèÅ The main() Function Pattern

Professional Code Organization

üéØ Best Practice: Use main() as the entry point of your program for better organization.

def greet(name):
    """Greet a person"""
    print(f"Hello, {name}!")

def calculate_area(radius):
    """Calculate circle area"""
    return 3.14159 * radius ** 2

def main():
    """Main program logic"""
    greet("Alice")
    area = calculate_area(5)
    print(f"Circle area: {area}")

# Entry point
if __name__ == "__main__":
    main()

üí° Why this pattern? It makes your code reusable as a module and keeps the entry point clear.

‚ú® List Comprehensions

# Transform strings to uppercase

üî∑ Sets: Unique Unordered Collections

Mathematical Sets in Python

# Creating sets
fruits = {'apple', 'banana', 'cherry'}
numbers = set([1, 2, 3, 2, 1])  # Duplicates removed!
print(numbers)  # {1, 2, 3}

# Adding and removing elements
fruits.add('date')
fruits.remove('apple')  # Error if not found
fruits.discard('kiwi')  # No error if not found

# Membership testing (very fast!)
print('banana' in fruits)  # True
# Set operations
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Union (all elements)
print(a | b)  # {1,2,3,4,5,6}

# Intersection (common)
print(a & b)  # {3, 4}

# Difference
print(a - b)  # {1, 2}

‚úÖ When to Use Sets

  • Remove duplicates
  • Fast membership testing
  • Mathematical operations
  • Find unique items

üîÑ Choosing the Right Collection

Quick Decision Guide

üìã Use Lists When:

  • Order matters
  • Need to modify data
  • Duplicates are okay
  • Need indexing/slicing
Most versatile!

üîí Use Tuples When:

  • Data shouldn't change
  • Need dictionary keys
  • Return multiple values
  • Represent fixed data
Immutable safety!

üìñ Use Dictionaries When:

  • Need key-value pairs
  • Fast lookups by key
  • Representing real objects
  • Counting/grouping data
Key-value power!

üî∑ Use Sets When:

  • Need unique values only
  • Fast membership testing
  • Set math operations
  • Order doesn't matter
Uniqueness guaranteed!
